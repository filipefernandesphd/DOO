# **Desafio: Jogo da VelIA**

## REGRAS

- Os desenvolvedores não devem mudar a estrutura do projeto, tampouco o arquivo `App.java`. A única modificação permitida é a inclusão das classes referentes aos jogadores (jogador 1 e jogador 2);
- O combate só será realizado se, e somente se, houverem, no mínimo, 2 desenvolvedores;
- As etapas para iniciar o combate são:
    - Usar integralmente a estrutura base (código-fonte) fornecida;
    - O projeto deve funcionar em qualquer máquina com JDK igual ou superior a versão 21;
    - A implementação deve demonstrar seu pleno funcionamento simulando o jogo entre duas instâncias de usários;
- Após a validação dos passos anteriores, o combate iniciará:
- Serão copiadas as pastas de cada desenvolvedor, cuja classe principal deve ser `Player.java`. **ATENÇÃO!** A classe `Player.java` deve estar contida em um pacote dentro da pasta `ia`. Por exemplo: `./ia/MinhaImplementacao/Player.java`.
- A implementação deve seguir os passos:
    - Ininicação do jogo;
    - Mostrar os nomes dos jogadores;
    - Atualizar o terminal a cada jogada;
    - O intervalo padrão de cada jogada deve ser de 2 segundos. Porém, deve ser possível mudá-lo;
    - Deve-se mostrar na tela o status atual do jogo. Por exemplo, "iniciando jogada...", "velha", "vencedor" etc.
- A implementação deve, **preferencialmente**, conter, no mínimo:
    - Algum dos 23 padrões de projeto GoF;
    - Tratamento de exceções.

## CRITÉRIOS DE AVALIAÇÃO

- A pontuação será de acordo com o ranking: 1º lugar - 4pts; 2º lugar - 2pts; 3º lugar - 1pt;
- **IMPORTANTE!** A pontuação só será concedida após arguição verbal. Caso o desenvolvedor demonstre conhecimento pleno sobre a implmentação e que não tenha usado geração de código por meio de ferramentas de IA, a pontuação será atribuída.
